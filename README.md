# Emerging Technologies Assessment — Summer 25/26

A Jupyter notebook exploring the difference between classical and quantum algorithms through five hands-on problems, centred around the [Deutsch–Jozsa algorithm](https://quantum.cloud.ibm.com/learning/en/modules/computer-science/deutsch-jozsa) — a quantum algorithm that demonstrates an exponential speedup over its classical counterpart.

## Problems

| # | Title | Description |
|---|-------|-------------|
| 1 | Generating Random Boolean Functions | Generate random constant or balanced Boolean functions with four inputs |
| 2 | Classical Testing for Function Type | Classically determine whether a function is constant or balanced, with an efficiency analysis |
| 3 | Quantum Oracles | Build quantum oracles for all single-input Boolean functions using Qiskit |
| 4 | Deutsch's Algorithm | Implement Deutsch's algorithm in Qiskit and demonstrate it on all four oracles |
| 5 | Deutsch–Jozsa Algorithm | Scale Deutsch's algorithm to handle four-input Boolean functions |

## Quick Start

The standard workflow to reproduce all results is:

```bash
git clone https://github.com/HaoYuYong/EmergingTechnologies-Assessment.git
cd EmergingTechnologies-Assessment
pip install -r requirements.txt
jupyter notebook problems.ipynb
```

Then select **Kernel → Restart & Run All** in the browser to run every cell in order.

## Detailed Setup

### Prerequisites

| Requirement | Version | Notes |
|---|---|---|
| [Python](https://www.python.org/downloads/) | 3.10 or later | Earlier versions are not supported |
| `pip` | any recent | Included with Python |
| A terminal / command prompt | — | PowerShell, bash, zsh, etc. |

You can confirm the installed Python version with:

```bash
python --version
```

### 1. Clone the repository

```bash
git clone https://github.com/HaoYuYong/EmergingTechnologies-Assessment.git
cd EmergingTechnologies-Assessment
```

### 2. (Optional) Create a virtual environment

> This step is **optional**. If you prefer to use your system Python directly,
> skip ahead to Step 3 — the install and launch commands work identically with
> or without a virtual environment.

A virtual environment isolates the project's dependencies from your system Python installation and avoids version conflicts with other projects.

```bash
python -m venv venv

# Activate on macOS / Linux
source venv/bin/activate

# Activate on Windows (PowerShell)
venv\Scripts\Activate.ps1

# Activate on Windows (Command Prompt)
venv\Scripts\activate.bat
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

This installs all required packages, including `qiskit`, `qiskit-aer`, `numpy`, `matplotlib`, and `jupyter`. Run this command whether or not you created a virtual environment in Step 2 — the install target adjusts automatically.

### 4. Launch the notebook

```bash
jupyter notebook problems.ipynb
```

Your default browser opens the notebook automatically.
Select **Kernel → Restart & Run All** to execute every cell from scratch and reproduce a clean output.

### Verifying the environment

If you want to confirm Qiskit is installed correctly before opening the notebook, run:

```bash
python -c "import qiskit; print(qiskit.__version__)"
```

## Verifying a Successful Run

After **Kernel → Restart & Run All**, every problem prints a table whose
final column flags whether the result matches the expected behaviour:

| Problem | What to look for |
|---|---|
| 1 | A 16-row truth table for one randomly generated 4-bit function |
| 2 | 10-trial classification table — every `Match` column reads `YES` |
| 3 | Oracle verification table — every `Pass` column reads `YES` |
| 4 | Deutsch's algorithm classification table — every `Pass` column reads `YES` |
| 5 | Deutsch–Jozsa classification table — every `Pass` column reads `YES` |

If every `Match` / `Pass` column shows `YES`, the notebook has executed
correctly end-to-end. The full run takes well under a minute on a modern
laptop.

## Troubleshooting

| Problem | Fix |
|---|---|
| `ModuleNotFoundError: No module named 'qiskit'` | Re-run `pip install -r requirements.txt` in the active environment. |
| Circuit diagrams do not render | Ensure `matplotlib` is installed and that you are using a Jupyter front-end (notebook or lab). |
| Wrong Python version | Confirm `python --version` reports 3.10 or later, or activate the virtual environment created in step 2. |
